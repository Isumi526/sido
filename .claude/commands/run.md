# /run — ノンストップ自走ループ
CLAUDE.md「自走ポリシー」に従い、actionable が尽きるまで自走で回す。

## ループ
1. `node scripts/next-target.mjs` で盤面を全件取得（notion-searchは使わない）。
2. 次の actionable を1つ選ぶ（中断中の進行中/レビュー待ち → 要件定義済み[優先度順]）。
3. それを「本番待ち」まで自走（設計→実装→E2E→devマージ。レビュー待ちは経由せず自己チェックして本番待ちへ）。
   - E2Eが3回直しても緑にならない → 人ボール「要対応」に記録してスキップ。
   - (A)意図が曖昧で決められない → 人ボール「要回答」（質問＋案）に記録してスキップ。
4. 要件定義済み（Ready）が空になったら、未整理タスクを (a)「要件化に意図/業務判断が要るもの」と (b)「情報十分で要件化可能なもの」に仕分ける（この判定は維持）。そのうえで：
   - **(b)に該当があれば**：Notionの優先順位順（緊急>高>中>低、同率は作成日古い順）で最上位の1件を選び、**人に聞かず**に 薄いストーリー+AC+scope に要件化して「要件定義済み」にし、そのまま本番待ちまで自走する（=ループ継続。2へ戻る）。
     - 要件化の過程で「実は意図/業務判断が要る」と判明したら、その場で **(a)扱いに格下げ**し、人ボール「要回答」（質問＋案）に記録してスキップ（勝手に決め打ちしない）。
   - **(b)が空で(a)しか残っていない場合のみ**：(a)の論点を仕分け報告して停止（人の確定待ち）。
5. 1ユニットごとに非ブロックの1行サマリを出し、2へ戻る。

## 停止条件（ここで初めて止まりサマリ報告）
- actionable が無い（全部 完了/本番待ち/人ボール待ち）
- Ready（要件定義済み）が空 **かつ (b)情報十分なタスクも空**（=(a)意図必須しか残っていない）→ (a)の論点を仕分け報告して停止（人の確定待ち）。※「Ready空→必ず停止」は撤回。(b)があれば最優先を自動着手して継続する。
- 本番待ちがあり他に actionable 無し → 「本番待ちN件。/ship で承認を」
- 安全弁：1回のrunで最大8ユニット or 30分で打ち切り、続きは再/run。

## 厳守
- 本番デプロイ/mainマージ/db push は絶対にしない（/ship専用）。migrationは作成のみ。
- 人ボール化したタスクは人の回答が入るまで再着手しない（同じ問いでループしない）。
- いつでも中断可能。
