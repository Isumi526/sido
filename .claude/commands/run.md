# /run — ノンストップ自走ループ
CLAUDE.md「自走ポリシー」に従い、actionable が尽きるまで自走で回す。

**人ボールを出す時は CLAUDE.md「人ボール報告の必須フォーマット（結論ファースト）」に従う**：画面出力は「🙋 [種別] タスク名／【決めてほしいこと】／【選択肢】A/B/C／【推奨】X」の構造で結論を先頭に。`--detail` は「A・B・Cどれ？推奨X」が分かる1〜2行に圧縮。

## ループ（CLAUDE.md「/run の優先順位決定」に従い、常に最優先1件を特定して前進）
判定対象は **`.env` の `BACKLOG_PROJECT_ID`（自案件）で絞った範囲**のみ。

1. `node scripts/next-target.mjs` で盤面を取得（案件で絞り済み・notion-searchは使わない）。
2. **最優先1件を特定**（上から順に該当する最初のもの）：
   - (1) 中断中の **進行中 / レビュー待ち** → 最優先で再開。
   - (2) 無ければ **要件定義済み（Ready）** を優先順位順（緊急>高>中>低、同率は作成日古い順）で先頭。
   - (3) Ready も無ければ **未整理（Inbox）** を優先順位順で先頭1件（→ 手順4で判定）。
3. 選んだ実装可能タスク（進行中/レビュー待ち/Ready）に**着手＝まず Notion ステータスを「進行中」に更新してから**設計→実装→E2E→devマージ と進める（中断中の進行中は既に「進行中」なのでそのまま／レビュー待ちは経由せず自己チェック）。実装・ローカル検証・devマージまで完了したら **「本番待ち」に更新**。完了したら **1 に戻る**。
   - 遷移は **要件定義済み →(着手)→ 進行中 →(実装完了)→ 本番待ち**。着手した瞬間に「進行中」にすることで Notion 上で『今 CC が実装中の1件』を可視化する（同時に進行中は原則1件）。
   - E2Eが3回直しても緑にならない → 人ボール「要対応」に記録してスキップ。記録後に LINE 通知：
     `node scripts/notify-humanball.mjs --kind 要対応 --task "<タスク名>" --detail "<失敗理由/エラー要約>" [--url "<セッションurl>"]`
   - (A)意図が曖昧で決められない → 人ボール「要回答」（質問＋案）に記録してスキップ。記録後に LINE 通知：
     `node scripts/notify-humanball.mjs --kind 要回答 --task "<タスク名>" --detail "<決めてほしいこと＋A/B/C＋推奨を1〜2行に圧縮>" [--url "<セッションurl>"]`
4. **Ready が空で 未整理（Inbox）を拾った場合**：その1件の内容を判定する（要件化の材料が本文にあるか）：
   - **(a) 意図・業務判断が必要**（要件/仕様が曖昧で人にしか決められない、**または本文が空/不足で要件化の材料が無い**）
     → 人ボール「要回答」として記録し、**実装はしない**。記録後に LINE 通知（質問＋案を添える）：
     `node scripts/notify-humanball.mjs --kind 要回答 --task "<タスク名>" --detail "<決めてほしいこと＋A/B/C＋推奨を1〜2行に圧縮>" [--url "<セッションurl>"]`
   - **(b) 情報十分で要件化できる**（ストーリー・AC・スコープを本文から起こせる）
     → **人に聞かず** に 薄いストーリー+AC+scope に要件化して **ステータスを「要件定義済み」に更新** → **実装に入る時に「進行中」に更新** → そのまま本番待ちまで自走する。
     - 要件化の過程で「実は意図/業務判断が要る」と判明したら、その場で **(a)扱いに格下げ**し、人ボール「要回答」に記録してスキップ（勝手に決め打ちしない）＋上記の LINE 通知。
   - **未整理を1件処理したら必ず 1 に戻る**（Ready が生まれるか、(a)で止まるか、未整理が尽きるまで繰り返す）。
5. 1ユニットごとに非ブロックの1行サマリを出し、1へ戻る。

## 停止条件（ここで初めて止まりサマリ報告）
- **Ready も 未整理（Inbox）も無い**（＝本当に手が無い）→ 停止して報告＋LINE通知：
  `node scripts/notify-humanball.mjs --kind 要回答 --task "Inbox枯渇：着手可能タスクなし" --detail "<盤面サマリ（本番待ちN件等）>" [--url "<セッションurl>"]`
- 着手可能（進行中/レビュー待ち/Ready）が無く、未整理が **(a)意図必須ばかり** → 各 (a) を人ボール「要回答」化して通知し続け、全て処理したら停止（人の確定待ち）。※「未着手だから検知できない」状態は作らない。
- 本番待ちがあり他に actionable 無し → 「本番待ちN件。/ship で承認を」
- 安全弁：1回のrunで最大8ユニット or 30分で打ち切り、続きは再/run。

## 厳守
- 本番デプロイ/mainマージ/db push は絶対にしない（/ship専用）。migrationは作成のみ。
- 人ボール化したタスクは人の回答が入るまで再着手しない（同じ問いでループしない）。
- いつでも中断可能。
