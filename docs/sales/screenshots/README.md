# 営業用スクリーンショット（撮影プラン＋一覧）

> **状態**: 撮影手順・推奨画面リストは確定済み。**実画像セットの確定は「デモデータ整備＋画面選定」という人の判断が要る**ため、ここでは撮影プランを置く（CC 2026-06-28）。
> **重要ルール（ユーザー指示）**: 本番URL/本番DBには一切接続しない。**デモ環境/ローカルのみ**で撮影する。

## 撮影手順（ローカル・デモ環境）

1. ローカルスタックを起動（`supabase start`）し、admin を起動（`cd apps/admin && npm run dev` → http://localhost:3001 ）。
2. テストアカウントでログイン済みの状態にする（ブランド表示=GENLINKS）。
3. **「営業映え」用のデモデータを入れる**（←ここが人の判断）。現状のE2Eシードは金額が小さく数も少ないので、スライド映えする現場数・金額・工程・見積を用意するのが望ましい。
4. 下記の各画面に遷移し、フルページでキャプチャ（ブラウザ拡張 or OSのスクショ）。`docs/sales/screenshots/` に保存。
