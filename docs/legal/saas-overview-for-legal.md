## 1. 利用者・データ主体の種別

| 種別 | 説明 | 主な利用面 |

## 3. データの流れ（収集→保存→利用→提供）

```
[作業員] LINE LIFF（日報/経費/勤怠/位置情報・領収書画像）
   │  (HTTPS / Supabase anon公開キー + 行レベルセキュリティ＝段階導入中)
   ▼
[Supabase]  PostgreSQL（個人データ本体） + Storage（領収書/PDF/添付）
   │
   ├─▶ [管理画面 admin]（管理者が認証ログインして閲覧・編集）
   │
   ├─▶ [Edge Functions]（サーバー処理）
   │       ├─ 未送信リマインド / 車検リマインド → LINE へ push通知
   │       ├─ 経費・注文書メール → Resend 経由でメール送信
   │       └─ 日報テキストのAI正規化 → Google Gemini API へ送信
   │
   └─▶ [Google Apps Script]（Phase1のLINE日報自動転記・一部レガシー）
```

- **位置情報**: 出退勤チェックイン時に取得し `attendance_logs` に保存。
- **領収書画像/PDF**: Supabase Storage に保存し、閲覧は短時間有効の署名URLで配信（公開URLではない）。
- **AI正規化**: 日報テキスト等を Google Gemini に送信して整形。【要確認: Gemini に送る内容に個人名等が含まれる前提で、AI委託先への提供を規約・PPに明記】
- **LINE通知**: リマインド等を作業員/管理者のLINEへ push。

## 4. 外部委託先（サブプロセッサ）と保存場所・国

| 委託先 | 役割 | データ所在地 |
|---|---|---|
| **Supabase**（DB/認証/ストレージ/Edge Functions） | 個人データ本体・画像の保存と処理 | リージョン **ap-northeast-2（ソウル・韓国）** 【要確認: 本番プロジェクトのリージョン。韓国保管＝越境移転の論点】 |
| **Vercel**（ホスティング） | LIFF/管理画面の配信（フロントエンド） | エッジ配信。【要確認: リージョン】 |
| **LINE（LINEヤフー）** | LIFFログイン基盤・push通知 | 日本 ほか |
| **Resend** | メール送信（経費申請・注文書承諾） | 米国 【要確認】 |
| **Google（Gemini API）** | 日報テキストのAI正規化 | 【要確認: リージョン・データ学習利用の有無（APIのデータ利用ポリシー）】 |
| **Google（Apps Script）** | Phase1レガシー処理 | 米国 ほか |

> **越境移転の主要論点**: 個人データ本体が **韓国（Supabaseソウルリージョン）** に保管される前提。外国にある第三者への提供／越境移転に関する同意・公表が必要になる可能性が高い。データ所在地は事業者が Supabase ダッシュボードで確定すること。

## 6. セキュリティ（現状の実装）

- 通信はHTTPS。管理画面はメール/パスワード認証（Supabase Auth）。
- 領収書・現場添付資料は **非公開バケット＋短時間署名URL** で配信（公開URLにしない）。
- 行レベルセキュリティ（RLS／テナント分離）は **段階導入中**。一部マスタ系テーブルは現状RLS未適用で、公開anonキー前提のデータ保護強化を別途進行中（社内タスクで対応中）。
- 下請け業者ページはトークン付きURL（期限・失効あり）でアクセス制御。

> 【要確認】規約・PPでは「適切な安全管理措置を講じる」旨を記載しつつ、過度に具体的な実装は約束しすぎない立て付けが望ましい（法務判断）。

## 7. 法務に確定してもらいたい論点（チェックリスト）

1. 取得する個人データの **利用目的** の特定・公表（特に位置情報・労務・経費）。
2. **位置情報（GPS）** の取得についての同意取得方法と目的明示。
3. **外国（韓国/米国）の第三者への提供・越境移転**（Supabase/Resend/Google）の同意・公表。
4. **AI（Gemini）への個人データ提供**の位置づけ（委託 or 第三者提供／学習利用の有無）。
5. 従業員（作業員）の個人データを **事業者（施工会社）が取り扱う** 構造の整理（テナント＝個人情報取扱事業者、当社＝委託先 という整理でよいか）。
6. データ **保持期間・削除** ポリシー。
7. 取引先（下請け）の **口座情報等** の取扱い。
8. 規約上の **責任制限・SLA・解約時データ取扱い・準拠法/管轄**。
