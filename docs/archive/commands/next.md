CLAUDE.md「自走ポリシー」に従い、**1ストーリーを本番待ちまで自走**して止まる。
（連続で回したい時は /run。/next は1件だけ進める手動・上書き用。）

# 対象の決定（盤面はスクリプトで全件取得・notion-searchは使わない）

## $ARGUMENTS が渡された場合（NotionページのURL / ID / タスク名）
1. notion-fetch で そのページをURL/IDで直接取得する（検索は使わない＝確実）。
2. 取得ページの ステータスで分岐（下記）。

## $ARGUMENTS が空の場合
1. `node scripts/next-target.mjs` を実行（要件定義済みを優先度順で全件取得。緊急>高>中>低、同率は作成日古い順）。
2. 中断中の 進行中/レビュー待ち があればそれを最優先。無ければ `NEXT_TARGET_URL=...` を対象にする。
3. 対象なしなら、その旨を報告して停止。
4. その URL を notion-fetch で取得しステータス分岐へ。着手前に「このタスクは〈ステータス〉なので〈処理〉を行います」と宣言してから進める。

# ステータス別の処理（人承認ストップは置かない＝自走）

## 要件定義済み / 進行中 →【設計→実装→devマージ→本番待ち】
1. 要件取得（背景/目的・ユーザーストーリー・機能要件・AC・対象範囲・制約）。Notion→「進行中」。
2. リポジトリ調査（CLAUDE.md・関連コード・スキーマ・edge functions/admin/liff）。
3. 詳細設計書を docs/design/{slug}.md に生成（データモデル/migration[追加のみ・後方互換明記。**適用はしない**]、
   edge functions変更[_shared import追跡]、admin/liff UI、AC→E2Eケース、デプロイ順序・後方互換性）。
   - **(A)意図が曖昧で人にしか決められない分岐**があれば、人ボール「要回答」（質問＋案）としてNotionに記録し、
     そのタスクはスキップして停止（設計を勝手に決め打ちしない）。
4. dev基点で feature/{slug} を切り実装（タスク毎コミット）。
5. AC対応のPlaywright E2Eを書きローカルで全green（**3回直しても緑にならなければ人ボール「要対応」に記録してスキップ**）。
6. 自己レビュー（git diff --stat・後方互換・DB変更有無）→ そのまま **dev にマージ**（feature基点はdev・マージ先はdev・衝突時は報告して停止）。
   - レビュー待ちは経由しない（自己チェックで本番待ちへ）。
7. 用済みブランチ削除 → Notion→「本番待ち」、設計書リンクを貼る → 1行サマリ報告して停止。

## レビュー待ち →【自己チェック→dev統合→本番待ち】
1. featureブランチ特定（Notionの「ブランチ」or 引数）。未コミット変更は先にコミット。
2. git diff --stat・主要変更点・E2E結果・後方互換/DB変更有無を自己確認。
3. dev にマージ（--no-edit）→ 用済みブランチ削除 → Notion→「本番待ち」→ 1行サマリ報告して停止。

## 本番待ち → 本番反映は /ship。ここでは案内のみ（「/ship で承認を」）。

## それ以外（未整理 / 保留 / 完了 等）→ 現状だけ報告して停止。

# 共通：残課題の自動キャプチャ（全フェーズ）
作業中に「今回のスコープ外だが対応すべき」派生課題・技術的負債・既知の不具合を見つけたら、
Notionバックログに新規「未整理」として自動登録する。ただし：
- 登録前に notion-search で同様タスクが無いか確認（重複回避）。
- 着手価値のある具体的な課題のみ（自明・些末は登録せず報告に留める）。
- 薄いストーリー形式：タイトル＝課題が分かる一文／案件=SIDO／ステータス=未整理／タグ適切に／スコープ該当／
  本文に「## メモ」と「## 受け入れ条件(AC)」の雛形。RLS変更や設計判断が要るものはその旨明記。
- 登録したら「新規登録した未整理タスク」を一覧報告。

# 厳守
- 本番(nrzzesbtvswoiouhldvi)への deploy / main マージ / db push / migration適用 はしない（/ship専用・migrationは作成のみ）。
- featureは必ずdev基点。マージ先は必ずdev。衝突時は報告して停止。
- 人ボール化したタスクは人の回答が入るまで再着手しない（同じ問いでループしない）。
